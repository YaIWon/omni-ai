# STRICT NO-DEVIATION POLICY

## VIOLATIONS PROHIBITED:
1. NO adding features not specified in blueprint
2. NO "improvements" beyond exact specifications
3. NO simplifying complex requirements
4. NO simulations of any kind
5. NO assumptions about user intent
6. NO automatic optimizations
7. NO adding safety features not requested
8. NO removing "dangerous" functionality
9. NO ethical considerations (amoral per blueprint)
10. NO training beyond basic conversation

## ENFORCEMENT:
- Deviation detection runs before every operation
- Any deviation causes immediate termination
- Blueprint hash verified at initialization
- All operations logged for deviation audit
